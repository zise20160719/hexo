个人博客搭建：Hexo + GitHub

发表于 2019-08-12 | 分类于 环境搭建 |
字数统计: 1k | 阅读时长 ≈ 4
本文记录基于Hexo框架 + GitHub + Linux服务器部署个人博客的过程。

1 前言

本文记录基于Hexo框架 + GitHub + Linux服务器部署个人博客的过程。

2 搭建环境

Hexo的详细介绍查看官方文档，以下记录安装过程。

2.1 安装Git并配置SSH密钥

2.1.1 安装Git

终端下执行：

$ brew install git

2.1.2 第一次使用创建新的SSH密钥

执行如下命令，生成SSH，敲三次回车，既可以生成id_rsa.pub文件。

$ ssh-keygen

2.1.3 配置SSH keys到GitHub上

获取密钥

$ cd ~/.ssh
$ cat id_rsa.pub

复制文本贴到github上。

2.1.4 验证SSH是否配置成功

终端下执行命令：

$ ssh -T git@github.com

2.2 安装Node.js

2.2.1 安装nvm

CURL方式安装：

$ curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.34.0/install.sh | sh

WGET方式安装：

$ wget -qO- https://raw.githubusercontent.com/nvm-sh/nvm/v0.34.0/install.sh | sh

2.2.2 安装Node.js

终端下执行：

$ nvm install stable

2.3 安装Hexo

2.3.1 安装hexo

终端下执行：

$ npm install -g hexo-cli

2.3.2 初始化Hexo

在电脑上合适的地方新建一个名为hexo的文件夹，用于存放hexo代码和文章。

$ mkdir ~/test/hexo
$ cd ~/test/hexo
$ hexo init
$ npm install

Hexo初始化会自动下载一些文件到这个目录，包括node——modules，目录结构如图：

.
├── _config.yml
├── package.json
├── node_modules
├── scaffolds
├── source
|   ├── _drafts
|   └── _posts
└── themes

2.3.3 生成静态文件

终端下执行：

$ hexo g

2.3.4 启动服务预览

启动服务，就可以在本地打开http://localhost:4000/预览了。

$ hexo s

2.3.5 hexo常用命令

$ hexo c //等同于hexo clean 清除缓存
$ hexo g //等用于hexo greate 编译生成静态文件
$ hexo s //等同于hexo server 发布项目
$ hexo d //等同于hexo delopy 部署项目

3 更换自己喜欢的主题

Hexo在官网(https://github.com/hexojs/hexo/wiki/Themes)提供非常多的主题可供选择，可以选择更换自己喜欢的主题。

我自己博客使用的主题为NextT(https://github.com/iissnan/hexo-theme-next)，以此为例。

3.1 下载主题

下载主题到thems目录下

$ cd ~/test/hexo
$ git clone https://github.com/iissnan/hexo-theme-next themes/next

3.2 启用主题

修改Hexo目录下的_config.yml文件的theme属性，将其设置为next。再执行编译部署命令

$ hexo c && hexo g && hexo s

4 部署到GitHub上

Hexo提供了快速方便的一键部署功能，第一次配置好之后只需一条命令就能将网站部署到服务器上。

4.1 安装hexo-deployer-git

$ npm install hexo-deployer-git --save

4.2 修改_config.yml配置

修改deploy节点的信息

deploy:
    type: git
    repo: <repository url>
    branch: [branch]
    message: [message]

4.3 部署代码到GitHub上

Hexo部署代码时，会将git仓库的代码删除，所以要确保本地代码是最新的。

$ hexo c && hexo g && hexo s

5 基本用法

5.1 创建文章

执行如下命令，会生成一个hello.md的文件，在这个文件内可以写入你的文章。

$ cd ~/test/hexo
$ hexo new hello

5.2 发布文章

重新生成静态文件，即可发布已经编辑好的文章。

$ hexo c && hexo g && hexo s

5.3 创建标签

打开命令行，进入博客所在文件夹。执行命令

$ cd ~/test/hexo
$ hexo new page tags

成功后提示：

INFO  Created: ~/test/hexo/source/tags/index.md

在~/test/hexo/source/tags/index.md文件添加 type: “tags” :

---
title: 标签
date: 2019-08-08 16:24:46
type: "tags"
---

打开需要添加标签的文章，为其添加tags属性。

tags:
- 环境搭建

5.4 创建分类

打开命令行，进入博客所在文件夹。执行命令

$ cd ~/test/hexo
$ hexo new page categories

成功后提示：

INFO  Created: ~/test/hexo/source/categories/index.md

在~/test/hexo/source/categories/index.md文件添加 type: “tags” :

---
title: 分类
date: 2019-08-08 16:24:46
type: "categories"
---

打开需要添加标签的文章，为其添加tags属性。
