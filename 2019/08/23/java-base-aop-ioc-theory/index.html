Java基础：AOP和IOC的底层原理

发表于 2019-08-23 |
字数统计: 408 | 阅读时长 ≈ 1
本文记录Java基础：AOP和IOC的底层原理。

1 IOC：Inversion of Control，指容器控制程序对象之间的关系。

1、控制反转：原来由程序本身去控制对象之间的依赖关系的格局被反转了，通过第三方容器(IOC)去完成这些对象的依赖关系并对他们进行集中管理。
2、依赖注入：获得依赖对象的过程由自身管理变为了有IOC容器自动注入，就是有IOC容器在运行期间，动态的将某种依赖关系注入对象之中。

2 AOP：Aspect oriented Programming，面向切面编程

1、AOP基于IOC，是对OOP的有益补充；
2、将业务模块所共同调用的逻辑或责任封装起来，比如日志记录，便于减少系统的重复代码，降低模块间的耦合度，并有利于未来的可操作性和可维护性。
3、实现原理：一、采用动态代理技术，利用截取消息的方式，对该消息进行装饰，以取代原有对象行为的执行；二、采用静态植入的方式，引入特定的语法创建切面，从而使得编译器可以再编译期间植入有关切面的代码。
4、使用场景：
Authentication 权限检查
Caching 缓存
Context passing 内容传递
Error handling 错误处理
Lazy loading 延迟加载
Debugging 调试
logging，tracing，profiling and monitoring 日志记录，跟踪，优化，校准
Persistence 持久化
Resource pooling 资源池
Synchronization 同步
Transactions 事务处理
另外Filter的实现和struts2的拦截器的实现都是AOP思想的提现。
